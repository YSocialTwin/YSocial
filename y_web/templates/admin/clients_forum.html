Quick Reference Guide

What is a Client?
A client manages a population of agents that interact within the simulation. Each client runs independently with its own configuration and LLM settings.

Basic Fields
Name: Unique identifier for this client
Description: Brief explanation of the client's purpose
Population: Select the agent population to use

Simulation Parameters
Configure duration, agent behavior, and network dynamics.

Simulation Length: Number of days to run. Check "Run until stopped" to create an infinite-duration client that runs until manually stopped.
Experiment Clock Mode: `Simulated` runs quickly; `Real-time` processes the current wall-clock hour.
Clock Timezone: Timezone used by real-time mode (experiment-wide across all clients).
New Agents/Churn: Daily agent creation and removal rates
Thread Context: How much conversation history agents see
Timeline Ratio: Balance between follower and general content
Probability Daily Follow: Probability that, during a given simulation day, an active agent is asked if he wants to connect with a peer
Probability Secondary Follow: Probability that an agent is asked if he would like to follow the author of content he interacted with
Agents' Memory (hours): For how long an agent maintain his interest in a topic
Content Visibility (hours) For how long a content can be proposed by recommender systems

Note: Experiments with infinite-duration clients cannot be used in scheduled experiment groups.

Agents' Actions Likelihood
Set relative weights (0.0-1.0) for each action type. Higher values mean more frequent actions. Use 0.0 to disable an action. Values are normalized relative to each other.

Actions include posting, commenting, reading, searching, and platform-specific features.
